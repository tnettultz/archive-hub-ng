# archive-hub-ng

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可包含信息展示、工具、文档或其他静态内容，不针对任何特定域名或网站。

## 项目简介

archive-hub-ng 是一个通用的 HTML 页面归档仓库。它旨在为多个不相关的静态页面提供一个集中存放和发布的场所，方便访问和索引。所有页面均以纯 HTML 形式存储，无需后端支持。

## 目录结构

```
archive-hub-ng/
├── pages/          # 存放所有归档的 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/         # 公共资源文件（如 CSS、JavaScript、图片等）
│   ├── css/
│   ├── js/
│   └── img/
├── index.html      # 仓库入口页面（可选）
└── README.md       # 本文件
```

- `pages/`：所有归档的 HTML 页面均放置在此目录下，可根据需要进一步划分子目录。
- `assets/`：存放页面共享的静态资源，保持文件结构清晰。
- `index.html`：可选的入口页面，用于导航或介绍。

## 页面归档说明

每个 HTML 页面应尽量独立、自包含。如需引用外部资源（如样式、脚本），建议使用相对路径指向 `assets/` 目录，或使用 CDN 链接。归档页面不限定主题或用途，但应避免包含恶意内容或违反相关法律法规。

### 添加新页面

1. 将新的 HTML 文件放入 `pages/` 目录。
2. 如页面需要专属资源，可在 `pages/` 下创建对应子目录，或统一使用 `assets/`。
3. 更新 `index.html`（如有）以添加新页面的链接。

### 页面命名规范

- 文件名使用小写字母和连字符，例如 `example-page.html`。
- 避免使用特殊字符或空格。

## 维护说明

本仓库由维护者定期管理。主要工作包括：

- 检查页面是否可正常访问和渲染。
- 清理不再需要的页面或资源。
- 更新 `index.html` 中的导航信息。

欢迎通过 Issues 或 Pull Requests 提交页面归档请求。提交前请确保页面内容符合仓库的基本要求。

## 许可

本仓库中的 HTML 页面及其资源文件，除非特别声明，均按各自页面的原始许可协议使用。仓库整体结构及 README 文件采用 [MIT 许可证](LICENSE)。
